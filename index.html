Access logs

Просмотреть log записи

access-log -r
access-log --request_id
Параметр Описание
-r STR --request_id STR Идентификатор log записи

Примеры:

# находит log запись
access-log --request_id e1e1j12j1123klkgygc567890

Chat

Просмотреть зарегистрированные чаты

chats

Зарегистрировать новый чат

chats new -c <int>
chats new --chat_id <int>
Параметр Описание
-c INT --chat_id INT Идентификатор чата в телеграм

Примеры:

# регистрирует чат
chats new --chat_id 1234567890

Удалить чат

chats delete -c <int>
chats delete --chat_id <int>
Параметр Описание
-c INT --chat_id INT Идентификатор чата в телеграм

Примеры:

# удаляет чат
chats delete --chat_id 1234567890

Startrek

Команды для взаимодействия с трекером задач

Просмотреть worklog записи

st worklog [-f <date>] [-t <date>]
startrek worklog [--from <date>] [--to <date>]
Параметр Описание
-f DATE --from DATE [Опциональный] При установке параметра from, будет выведен worklog за указанную дату. Формат ввода: YYYY-MM-DD
-t DATE --to DATE [Опциональный] При установке параметра to в связке с from, будет выведен worklog за промежуток [from, to). Формат ввода: YYYY-MM-DD

Примеры:

# вывод записей worklog за сегодня
st worklog

# вывод записей worklog за 2022-12-15
startrek worklog -f 2022-12-15

# вывод записей worklog за 2022-12-15/2022-12-17
st worklog --from 2022-12-15 --to 2022-12-18

Удалить worklog запись

st worklog-delete -i <str> -w <int>
startrek worklog-delete --issue_key <str> --worklog_id <int>
Параметр Описание
-i STR --issue_key STR Ключ задачи в трекере
-w INT --worklog_id INT Идентификатор записи в worklog

Примеры:

# удаляет запись worklog
st worklog-delete -i CLOUD-12345 --worklog_id 587429134

Создать worklog запись

st worklog-create -i <str> -d <str> [-s <datetime>] [-c <str>]
startrek worklog-create --issue_key <str> --duration <str> [--start <datetime>] [--comment <str>]
Параметр Описание
-i STR --issue_key STR Ключ задачи в трекере
-d STR --duration STR Затраченное время. Формат ввода [>0]w[>0]d[>0]m[>0]s
-s DATETIME --start DATETIME [Опциональный] Время начала работы над задачей. По умолчанию устанавливается как разность времени сейчас и продолжительности работы. Формат ввода YYYY-MM-DDThh:mm
-c STR --comment STR [Опциональный] Комментарий к записи в worklog. По умолчанию ничего не устанавливатеся

Примеры:

# создает запись worklog
startrek worklog-create -i CLOUD-12345 --duration 5h

# создает запись worklog с комментарием
st worklog-create -i CLOUD-12345 -d 5h --comment Ковырял в носу и ничего не сделал

# создает запись worklog с указанием времени начала
st worklog-create --issue_key CLOUD-12345 -d 5h -s 2022-12-15T10:00
